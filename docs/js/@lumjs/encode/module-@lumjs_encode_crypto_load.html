(static) hash(name, instanceopt) → {function|object}

Get a Hashing function or progressive instance.

Parameters:
Name Type Attributes Default Description
name string

The name of the algorithm.

Hashing algorithms supported by crypto-js:

  • MD5
  • SHA-1
  • SHA-256
  • SHA-512
  • SHA-224
  • SHA-384
  • SHA-3
  • RIPEMD-160

You can omit the - characters, and the parameter is case-insensitive.
instance boolean | object <optional>
 false

Return an instance?

If this is true we'll return an object instance for progressive hashing.

If this is an object, we'll return an object instance, and this will be passed as the options to the algo.create() method.

If this is false (default), we'll return a hashing function.

(static) load(name, toLoweropt) → {object|function}

Load and return a specific crypto-js module.

This is a wrapper for require() that prefixes the crypto-js/ package namespace.

Generally you should never have to call this manually. Instead use one of the more specialized methods.

Note that this is the actual default export from the load module, and all the other functions are child properties of it.

Parameters:
Name Type Attributes Default Description
name string

The name of the module.
toLower boolean <optional>
 true

Enforce lowercase names?

If true (the default value), the name will be normalized to lowercase as is expected by the library.
Source:
  • crypto/load.js, line 43
Returns:

The module content.

Some crypto-js modules return a child object, some return a child function, and some return the crypto-js/core object. The wide difference in return values is part of why this method should not be used directly.
